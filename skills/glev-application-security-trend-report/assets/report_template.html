About this report

The baseline scan captures all security findings present at the commit immediately before the audit window. Each subsequent month is scanned only over the files changed since the previous month-end commit. A finding is considered "newly introduced" in month N when its fingerprint did not appear in the baseline or any previous audited month.

Please note that the trend chart does not account for fixed findings: the "total findings" line is cumulative and only ever grows, so it reflects everything introduced over time rather than what remains open today.
